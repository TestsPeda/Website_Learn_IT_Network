Server-Betriebssysteme

A

Aufgaben & Abgrenzung zum Desktop-BS

Was ein Server-Betriebssystem leistet und worin es sich von einem Desktop-System unterscheidet.

01Multiple ChoiceAuto

Was ist die grundlegende Aufgabe eines Servers — bis heute unverändert?

  • Möglichst bunte Oberflächen für Endanwender darzustellen.
  • Auf die Anforderung eines Clients zu antworten und Ressourcen bereitzustellen.
  • Ausschließlich Spiele und Multimedia abzuspielen.
  • Sich nach Feierabend automatisch herunterzufahren.
Lösung

Schon „Windows for Workgroups“ machte Rechner per Definition zu Servern, weil sie Ressourcen weitergeben und auf Anforderungen von Clients antworten konnten. Bis heute gilt: ein Server ist ein System, das auf die Anforderung eines Clients antwortet.

02Multiple ChoiceAuto

Welche Eigenschaft trifft am ehesten auf ein Server-BS, aber nicht auf ein typisches Desktop-BS zu?

  • Es kann Textdateien öffnen.
  • Es besitzt einen Dateimanager.
  • Es ist auf 24/7-Dauerbetrieb und hohe Stabilität für viele gleichzeitige Nutzer ausgelegt.
  • Es benötigt eine Tastatur.
Lösung

Server-Betriebssysteme sind für Dauerbetrieb, Stabilität und viele gleichzeitige Verbindungen optimiert. Ein Desktop-BS wird dagegen typischerweise von einer Person genutzt und regelmäßig ausgeschaltet.

03Multi-SelectAuto

Welche Merkmale sind typisch für ein Server-BS? (Mehrfachauswahl)

  • Bereitstellung von Diensten/Rollen (Web, Datei, DHCP, DNS …).
  • Multiuser-/Mehrbenutzerbetrieb mit Rechte- und Rollenverwaltung.
  • Remote-Verwaltung, häufig ohne grafische Oberfläche (Core/CLI).
  • Fokus auf Spiele-Performance und Multimedia-Komfort.
  • Ausgelegt auf Stabilität und Dauerbetrieb (Uptime).
Lösung

Dienste/Rollen, Multiuser-Betrieb, Remote-Verwaltung und Dauerbetrieb kennzeichnen ein Server-BS. Verwechslung: Spiele-/Multimedia-Komfort ist ein Desktop-Thema, kein Serverziel.

04Multiple ChoiceAuto

Warum verzichten Server-Installationen oft auf eine grafische Oberfläche und laufen als Core- bzw. CLI-Variante?

  • Weil Server keine Festplatten haben.
  • Weniger laufende Komponenten = geringere Angriffsfläche, weniger Updates/Reboots und mehr Ressourcen für Dienste.
  • Weil eine GUI auf Servern technisch unmöglich ist.
  • Weil CLI langsamer ist und so Strom spart.
Lösung

Eine schlanke Core/CLI-Installation hat weniger Komponenten: kleinere Angriffsfläche, weniger Sicherheitsupdates und Neustarts, mehr RAM/CPU für die eigentlichen Dienste. Verwaltet wird remote, z. B. per SSH oder PowerShell.

B

Vertreter: Windows Server, Linux/Unix, NetWare

Die wichtigsten Server-Betriebssysteme aus dem Infotext und ihre Einordnung.

05Multiple ChoiceAuto

Welches war Microsofts erstes echtes Serverbetriebssystem?

  • Windows for Workgroups
  • Windows NT (Versionen 3.5 und 3.51)
  • Windows Server 2025
  • MS-DOS
Lösung

Microsofts erstes echtes Serverbetriebssystem war Windows NT (3.5 / 3.51). Darauf folgte die bis heute existierende Windows-Server-Reihe; die neueste Version im Infotext ist Windows Server 2025.

06Multiple ChoiceAuto

Welche der genannten Systeme gehören zur Linux-/Unix-Welt?

  • Red Hat Enterprise Linux, Debian und CentOS
  • Windows NT, Windows Server 2025
  • Azure, AWS, IBM Cloud
  • MS-DOS und Windows for Workgroups
Lösung

Zu den verbreiteten Linux-/Unix-Varianten zählen Red Hat Enterprise Linux (RHEL), Debian und CentOS. Linux ist als Open Source besonders als Webserver beliebt, oft mit dem Apache-Webanwendungsserver.

07Multiple ChoiceAuto

Welche Software wird „oft installiert“, weil Linux als Webserver so beliebt ist?

  • Der Hyper-V-Hypervisor
  • Der Apache-Webanwendungsserver
  • Novell Open Enterprise Server
  • Microsoft Azure
Lösung

Auf Linux-Webservern wird häufig der Apache-Webanwendungsserver eingesetzt — er liefert Webinhalte aus und ist selbst Open Source.

08Multiple ChoiceAuto

Was geschah schließlich mit NetWare?

  • Es wurde zu Windows NT weiterentwickelt.
  • Es ist heute das meistgenutzte Server-BS.
  • Es wurde auf einen Linux-basierten Kernel verlegt und in Novell Open Enterprise Server (OES) umbenannt.
  • Es wurde zu Amazons AWS-Plattform.
Lösung

NetWare wird nicht mehr produziert. Novell verlegte das Serverbetriebssystem auf einen Linux-basierten Kernel und nannte es Novell Open Enterprise Server (OES).

C

Lizenzmodelle: proprietär vs. Open Source

Wie sich Windows Server und Linux/Unix lizenzrechtlich unterscheiden.

10Multiple ChoiceAuto

Wie wird Linux im Infotext bezüglich seines Lizenzmodells beschrieben?

  • Als proprietäres System mit geschlossenem Quellcode.
  • Als Open-Source-Betriebssystem.
  • Als reines Cloud-Abo ohne lokale Installation.
  • Als kostenpflichtige Microsoft-Lizenz.
Lösung

Linux ist ein Open-Source-Betriebssystem. Der Quellcode ist offen einsehbar, das System frei nutzbar und anpassbar — ein Grund für die Beliebtheit als Webserver.

11Multi-SelectAuto

Welche Aussagen zu proprietär vs. Open Source sind korrekt? (Mehrfachauswahl)

  • Bei proprietären Systemen (z. B. Windows Server) ist der Quellcode geschlossen und die Nutzung an eine Lizenz gebunden.
  • Bei Open Source ist der Quellcode offen und das System darf angepasst und weitergegeben werden.
  • Open Source bedeutet immer, dass es keinerlei Support gibt.
  • Linux-Distributionen wie Debian oder CentOS sind typische Open-Source-Vertreter.
  • Windows Server ist Open Source.
Lösung

Proprietär = geschlossener Code + Lizenzbindung (Windows Server). Open Source = offener, anpassbarer Code (Linux, z. B. Debian/CentOS). Verwechslung: Auch für Open Source gibt es Support (Community oder kommerziell, z. B. Red Hat); und Windows Server ist nicht Open Source.

D

Rollen, Dienste & Virtualisierung

Wozu ein Server-BS eingesetzt wird und wie ein Hypervisor virtuelle Server ermöglicht.

12Multiple ChoiceAuto

Was unterstützt Windows Server 2025 unter anderem?

  • Nur ältere DOS-Programme.
  • Zahlreiche Anwendungen und Datenbanken sowie einen Hypervisor, der virtuelle Server ermöglicht.
  • Ausschließlich das Abspielen von Videos.
  • Nur einen einzigen gleichzeitigen Nutzer.
Lösung

Windows Server 2025 unterstützt zahlreiche Anwendungen und Datenbanken sowie einen Hypervisor (bei Microsoft Hyper-V), der virtuelle Server bereitstellt.

13Multiple ChoiceAuto

Was ist ein Cloudserver nach der Definition des Infotextes?

  • Ein Server, der nur lokal im eigenen Keller steht.
  • Ein virtueller Server, der auf der Infrastruktur eines Drittanbieters in einem offenen Netz wie dem Internet gehostet wird.
  • Ein Server ohne jegliches Betriebssystem.
  • Eine besondere Festplatte für Backups.
Lösung

Cloudserver sind virtuelle Server auf der Infrastruktur eines Drittanbieters (Google Cloud, Microsoft Azure, IBM Cloud). Vorreiter war Amazons AWS: Kunden erstellen nahezu augenblicklich einen virtuellen Server und passen die Ressourcen nach Bedarf an (Skalierung).

14Szenariomanuell

Ein kleines Unternehmen will mit knappem Budget mehrere Dienste (Webserver, Dateifreigabe, interne Datenbank) auf einer physischen Maschine betreiben und dabei lizenzkostenarm bleiben. Welche Entscheidungen bei Server-BS, Lizenzmodell und Virtualisierung empfiehlst du — und warum?

Musterlösung

Sinnvolle Kombination:

  • Server-BS: Eine Linux-Distribution (z. B. Debian oder eine RHEL-Variante), weil sie als Open Source lizenzkostenarm und sehr stabil im Dauerbetrieb ist.
  • Webserver: Den freien Apache-Webanwendungsserver — passt zur Linux-Welt und ist ebenfalls kostenfrei.
  • Mehrere Dienste auf einer Maschine: Über einen Hypervisor (Virtualisierung) lassen sich Web-, Datei- und Datenbankdienst in getrennten virtuellen Servern isolieren — das erhöht Stabilität und Sicherheit, da ein Problem nicht alle Dienste mitreißt.
  • Verwaltung: Schlanke Core/CLI-Installation und Remote-Verwaltung (z. B. per SSH) sparen Ressourcen und reduzieren die Angriffsfläche.
Alternative mit Microsoft-Welt: Windows Server mit Hyper-V bietet dieselbe Virtualisierung, ist aber proprietär und kostenpflichtig — daher bei knappem Budget zweite Wahl, dafür mit gewohnter GUI und Hersteller-Support.
