#16

E-Mail-Protokolle — SMTP, IMAP, POP3

Wie eine E-Mail vom Sender zum Empfänger gelangt: SMTP versendet, IMAP verwaltet serverseitig und synchron, POP3 lädt lokal herunter – jeweils unverschlüsselt oder per SSL/TLS abgesichert.

16 Aufgaben 3 Diagramme MC · Multi-Select · Zuordnung · Szenario
A

Der Weg einer E-Mail

Versand mit SMTP über Mailserver (MTA), Ablage im Postfach und Abruf mit POP3 oder IMAP.

01Multiple ChoiceAuto

Welches Protokoll wird zum Versenden von E-Mails verwendet?

  • IMAP
  • POP3
  • SMTP
  • HTTP
Lösung

SMTP (Simple Mail Transfer Protocol) ist für den Versand zuständig und transportiert die Nachricht vom Client zum Server des Empfängers. Verwechslung: IMAP und POP3 dienen nur dem Abruf.

02Multiple ChoiceAuto

Was bezeichnet die Relay-Funktion von SMTP?

  • Das automatische Löschen der Mail nach dem Abruf.
  • Das Weiterleiten der Nachricht über mehrere beteiligte Server bis zum Empfänger-Server.
  • Das Synchronisieren der Mail auf mehreren Endgeräten.
  • Das Verschlüsseln der Verbindung per STARTTLS.
Lösung

Beim Relay leitet ein SMTP-Server die Nachricht – wenn sie nicht direkt zugestellt werden kann – über einen oder mehrere weitere Relay-Server zum empfangenden Server weiter.

03Multiple ChoiceAuto

Was ist ein MTA (Mail Transfer Agent) im E-Mail-Weg?

  • Das Mailprogramm auf dem Smartphone des Nutzers.
  • Ein Mailserver, der Nachrichten per SMTP entgegennimmt und weitertransportiert.
  • Ein Protokoll zum Abruf von Mails.
  • Der lokale Posteingang-Ordner im Mail-Client.
Lösung

Ein MTA ist ein Mailserver, der eine Mail per SMTP annimmt und – ggf. über Relays – Richtung Empfänger weiterleitet. Das Mailprogramm des Nutzers ist dagegen der Client (MUA).

04Szenariomanuell

Beschreibe in eigenen Worten den Weg einer E-Mail von Anna an Ben – von Annas Mailprogramm bis Ben die Mail liest. Nenne dabei, welches Protokoll in welchem Schritt arbeitet.

Musterlösung
  1. Anna schreibt die Mail in ihrem Mail-Client und sendet sie ab.
  2. Der Client übergibt die Mail per SMTP an Annas Mailserver (sendender MTA).
  3. Dieser stellt die Mail per SMTP direkt oder über weitere Relay-Server an Bens Mailserver zu.
  4. Bens Mailserver legt die Nachricht in Bens Postfach ab.
  5. Ben ruft die Mail mit seinem Client per IMAP (serverseitig, synchron) oder POP3 (Download aufs Gerät) ab und liest sie.

Merksatz: SMTP für den Versand/Transport, POP3 oder IMAP für den Abruf.

B

POP3 vs. IMAP

Lokaler Download mit POP3 gegenüber serverseitiger, synchroner Verwaltung mit IMAP.

05Multiple ChoiceAuto

Was passiert standardmäßig mit E-Mails, die über POP3 abgerufen werden?

  • Sie bleiben auf dem Server.
  • Sie werden nach dem Herunterladen vom Server gelöscht.
  • Sie werden auf alle Geräte synchronisiert.
  • Sie werden auf dem Server archiviert.
Lösung

In der Standardkonfiguration lädt POP3 die Mail auf das Endgerät herunter und löscht sie anschließend vom Server. Danach wird die Nachricht nur noch lokal verwaltet.

06Multiple ChoiceAuto

Worin liegt der zentrale Vorteil von IMAP gegenüber POP3?

  • IMAP ist auch zum Versenden von Mails geeignet.
  • IMAP benötigt keine Internetverbindung.
  • Die Mails bleiben auf dem Server und werden über mehrere Geräte hinweg synchron verwaltet.
  • IMAP löscht die Mails sofort vom Server.
Lösung

Bei IMAP verbleiben die Mails (und Ordner) auf dem Server. Jede Aktion – Lesen, Verschieben, Löschen – wird serverseitig ausgeführt, sodass alle Geräte denselben Stand synchron sehen.

07Multi-SelectAuto

Welche Aussagen über IMAP sind richtig? (Mehrfachauswahl)

  • Mails werden zentral auf dem Server verwaltet.
  • Ordner können direkt am E-Mail-Server angelegt werden.
  • Aktionen wie Lesen, Verschieben und Löschen werden über alle Geräte synchronisiert.
  • Die Mails werden nach dem Abruf standardmäßig vom Server gelöscht.
  • IMAP ersetzt SMTP beim Versenden von Mails.
Lösung

Richtig sind die drei serverbasierten Eigenschaften: zentrale Verwaltung, serverseitige Ordner und Synchronisation über alle Geräte. Das Löschen nach Abruf ist POP3-Verhalten; das Versenden übernimmt SMTP.

08Zuordnungmanuell

Ordne jedem Protokoll seine Hauptaufgabe zu: SMTP, IMAP, POP3 ↔ (a) lokaler Download, löscht meist vom Server · (b) Versand/Transport · (c) serverseitige, synchrone Verwaltung.

Musterlösung
ProtokollHauptaufgabe
SMTP(b) Versenden / Transport der Mail zum Empfänger-Server (inkl. Relay)
IMAP(c) Serverseitige, synchrone Verwaltung der Mails über mehrere Geräte
POP3(a) Lokaler Download aufs Endgerät, löscht die Mail standardmäßig vom Server
09Szenariomanuell

Eine Nutzerin will ihre Mails auf Smartphone, Tablet und PC jederzeit mit demselben Stand sehen. Welches Abruf-Protokoll empfiehlst du und warum? Wann wäre umgekehrt POP3 sinnvoll?

Musterlösung

Empfehlung: IMAP. Da die Mails auf dem Server bleiben und jede Aktion serverseitig ausgeführt wird, finden alle drei Geräte stets denselben synchronen Status vor – ideal für die Mehrgeräte-Nutzung.

POP3 ist dagegen sinnvoll, wenn Mails bewusst lokal auf nur einem Gerät (oder im lokalen Netz) verwaltet werden sollen – etwa um sie vom Server zu entfernen oder offline zu organisieren. Eine Synchronisierung gibt es dann nicht.

C

Ports & Verschlüsselung

Standardports der drei Protokolle und die Absicherung per SSL/TLS.

10Multiple ChoiceAuto

Welcher Port ist der Standardport für unverschlüsseltes SMTP?

  • Port 110
  • Port 25
  • Port 143
  • Port 993
Lösung

Der Standardport für unverschlüsseltes SMTP ist 25. Verschlüsselt kommt klassisch 465 zum Einsatz (bzw. 587 für die Mail-Einreichung mit STARTTLS).

11Multiple ChoiceAuto

Welche Zuordnung der unverschlüsselten Standardports ist korrekt?

  • SMTP 143 · POP3 25 · IMAP 110
  • SMTP 25 · POP3 110 · IMAP 143
  • SMTP 110 · POP3 143 · IMAP 25
  • SMTP 465 · POP3 995 · IMAP 993
Lösung

Unverschlüsselt gilt: SMTP 25, POP3 110, IMAP 143. Die letzte Option nennt dagegen die verschlüsselten Ports (465/995/993).

12Multi-SelectAuto

Welche Port-Angaben gehören zu einer verschlüsselten (SSL/TLS) Verbindung? (Mehrfachauswahl)

  • 465 – SMTP verschlüsselt
  • 993 – IMAP verschlüsselt
  • 995 – POP3 verschlüsselt
  • 25 – SMTP
  • 143 – IMAP
Lösung

Verschlüsselt sind 465 (SMTP), 993 (IMAP) und 995 (POP3). Die Ports 25 und 143 sind die unverschlüsselten Pendants.

13Multiple ChoiceAuto

Was unterscheidet impliziten von explizitem TLS (z. B. STARTTLS)?

  • Implizit: Verbindung ist von Beginn an verschlüsselt (eigener Port). Explizit/STARTTLS: zunächst Klartext, dann Umschalten auf TLS über denselben Port.
  • Implizit verschlüsselt nur den Anhang, explizit nur den Betreff.
  • Beide sind identisch und nur unterschiedliche Namen.
  • Explizit ist unverschlüsselt, implizit verzichtet ganz auf Verschlüsselung.
Lösung

Bei implizitem TLS ist die Verbindung von Anfang an über einen eigenen, verschlüsselten Port aufgebaut (z. B. 465, 993, 995). Bei explizitem TLS startet die Verbindung im Klartext und wird per STARTTLS-Kommando auf demselben Port (z. B. 587, 143) auf TLS umgeschaltet.

14Multiple ChoiceAuto

Warum sollte man E-Mail-Verbindungen per SSL/TLS verschlüsseln?

  • Damit Mails schneller versendet werden.
  • Damit Zugangsdaten und Inhalte auf dem Transportweg nicht mitgelesen werden können.
  • Damit Mails automatisch auf alle Geräte synchronisiert werden.
  • Weil unverschlüsselte Protokolle keine Anhänge unterstützen.
Lösung

Bei unverschlüsselten Verbindungen (25, 110, 143) werden Anmeldedaten und Mailinhalte im Klartext übertragen und könnten abgefangen werden. SSL/TLS sichert den Transportweg ab.

15Lückentextmanuell

Vervollständige: „Das ____ -Protokoll ist für den Versand verantwortlich und überträgt Nachrichten über mehrere ____ (Relay-Funktion). Der unverschlüsselte Standardport ist ____ . Beim Abruf lädt ____ die Mails lokal herunter (Port ____ ), während ____ sie serverseitig synchron verwaltet (Port ____ )."

Musterlösung

Das SMTP-Protokoll ist für den Versand verantwortlich und überträgt Nachrichten über mehrere Server (Relay-Funktion). Der unverschlüsselte Standardport ist 25. Beim Abruf lädt POP3 die Mails lokal herunter (Port 110), während IMAP sie serverseitig synchron verwaltet (Port 143).

Eine Nutzerin möchte ihre E-Mails synchron auf Smartphone und Laptop lesen (gleicher Stand, gelesene Mails überall gelesen). Welches Abrufprotokoll empfiehlst du? Begründe und grenze zu POP3 ab. Nenne außerdem das Versandprotokoll und je einen typischen verschlüsselten Port.

Musterlösung
  • IMAP — die Mails bleiben auf dem Server, alle Geräte sehen denselben Stand (Ordner, gelesen/ungelesen synchron).
  • Abgrenzung POP3: lädt Mails herunter (oft mit Löschen auf dem Server) → nur auf einem Gerät sinnvoll, keine Synchronisation.
  • Versand über SMTP. Verschlüsselte Ports: IMAPS 993, SMTP 587 (STARTTLS) bzw. 465 (SMTPS).
  • IMAP empfohlen
  • Begründung Synchronisation
  • POP3 abgegrenzt
  • SMTP + Ports
