A

Seriell & Parallel

Wie werden die Bits über die Leitung(en) geschickt — nacheinander oder gleichzeitig?

01Multiple ChoiceAuto

Was kennzeichnet die serielle Übertragung?

  • Bits werden nacheinander über eine einzige Leitung übertragen.
  • Jedes Bit bekommt eine eigene Frequenz auf demselben Kabel.
  • Es können nur ganze Bytes, aber keine einzelnen Bits übertragen werden.
  • Mehrere Bits werden gleichzeitig über mehrere Leitungen übertragen.
Lösung

Seriell heißt sequenziell: Bits laufen nacheinander über eine Leitung. Vorteile: einfacher Aufbau, geringerer Hardwareaufwand, gut für lange Strecken, keine Laufzeitunterschiede.

02Multiple ChoiceAuto

Welches Problem tritt bei der parallelen Übertragung über mehrere Leitungen auf?

  • Die Übertragung ist grundsätzlich langsamer als seriell.
  • Es kann immer nur in eine Richtung gesendet werden.
  • Laufzeitunterschiede und Übersprechen – Bits können zeitlich versetzt ankommen.
  • Es lässt sich nur über Glasfaser realisieren.
Lösung

Parallel überträgt mehrere Bits gleichzeitig über mehrere Leitungen — hoher Durchsatz auf kurzen Strecken, aber aufwendige Verdrahtung, anfällig für Übersprechen und Laufzeitunterschiede (Bits kommen zeitlich versetzt an).

03Multiple ChoiceAuto

Welches ist ein Beispiel für eine serielle Schnittstelle?

  • IDE/PATA
  • LPT/Centronics-Druckerschnittstelle
  • USB
  • PCI-Bus
Lösung

USB ist seriell. Weitere serielle Beispiele: RS-232/UART (COM), SATA, SPI, I²C, Ethernet (intern seriell). Verwechslung LPT/Centronics, IDE/PATA und PCI-Bus sind parallel.

B

Richtungsbetrieb

Simplex, Halbduplex und Vollduplex — wer darf wann in welche Richtung senden?

04Multiple ChoiceAuto

Bei welcher Betriebsart ist nur eine Übertragungsrichtung möglich (kein Rückkanal)?

  • Multiplex
  • Halbduplex
  • Simplex
  • Vollduplex
Lösung

Simplex (SX) überträgt nur in eine Richtung; der Empfänger kann nicht zurücksenden. Beispiele: Rundfunk (UKW, DAB+), Fernsehen (DVB-T, Satellit), Pager.

05Multiple ChoiceAuto

Ein Walkie-Talkie – bei dem immer nur einer sprechen kann – ist ein Beispiel für …?

  • Simplex
  • Vollduplex
  • Frequenzmultiplex
  • Halbduplex
Lösung

Halbduplex (HX/HDX): bidirektional, aber nur eine Richtung gleichzeitig — Sender und Empfänger wechseln sich ab. Weitere Beispiele: WLAN (CSMA/CA), Ethernet mit Hub.

06Multiple ChoiceAuto

Welche Betriebsart erlaubt gleichzeitiges Senden und Empfangen (z. B. Telefon, Ethernet mit Switch)?

  • Simplex
  • Halbduplex
  • Codemultiplex
  • Vollduplex
Lösung

Vollduplex (DX/FDX): gleichzeitige Übertragung in beide Richtungen über getrennte Kanäle. Beispiele: Telefon/VoIP, Ethernet mit Switch (Vollduplex-Port), DSL/Glasfaser.

07Multiple ChoiceAuto

Als welche Betriebsart arbeitet klassisches WLAN (eine Funkfrequenz, Zugriff per CSMA/CA)?

  • Halbduplex
  • Wellenlängenmultiplex
  • Vollduplex
  • Simplex
Lösung

Klassisches WLAN nutzt eine gemeinsame Funkfrequenz und regelt den Zugriff per CSMA/CA — es kann immer nur eine Station gleichzeitig senden, also Halbduplex.

08Zuordnungmanuell

Ordne Simplex, Halbduplex und Vollduplex jeweils einer Definition und einem Beispiel zu.

Musterlösung
  • Simplex – Übertragung nur in eine Richtung, kein Rückkanal. Beispiel: Rundfunk (UKW/DAB+), Pager.
  • Halbduplex – beide Richtungen möglich, aber nur abwechselnd (nicht gleichzeitig). Beispiel: Walkie-Talkie, WLAN, Ethernet mit Hub.
  • Vollduplex – beide Richtungen gleichzeitig über getrennte Kanäle. Beispiel: Telefon/VoIP, Ethernet mit Switch, DSL.
C

Multiplexverfahren

Mehrere Signale teilen sich ein Medium — aufgeteilt nach Frequenz, Zeit, Code oder Wellenlänge.

09Multiple ChoiceAuto

Was bedeutet „Multiplexing"?

  • Mehrere Signale/Kanäle teilen sich dasselbe Übertragungsmedium.
  • Daten werden vor dem Senden komprimiert.
  • Bits werden parallel über mehrere Leitungen übertragen.
  • Ein Signal wird zur Sicherheit über mehrere getrennte Medien geschickt.
Lösung

Beim Multiplexing teilen sich mehrere Signale/Kanäle dasselbe Übertragungsmedium — getrennt nach Frequenz (FDM), Zeit (TDM), Code (CDM) oder Wellenlänge (WDM).

10Multiple ChoiceAuto

Nach welchem Kriterium teilt FDM (Frequenzmultiplex) das Medium auf?

  • Nach Frequenz (jedes Signal ein eigenes Frequenzband).
  • Nach Zeit (jedes Signal ein eigener Zeitschlitz).
  • Nach Code (jedes Signal ein eigener Spreizcode).
  • Nach Lichtwellenlänge (jedes Signal eine eigene Farbe).
Lösung

FDM teilt nach Frequenz: jedes Signal erhält ein eigenes Frequenzband, alle senden gleichzeitig. Medium: Funk/Kabel. Beispiele: UKW-Radio, Kabelfernsehen, DSL.

11Multiple ChoiceAuto

Wie funktioniert TDM (Zeitmultiplex)?

  • Gleiche Frequenz, jedes Signal sendet in einem eigenen, rotierenden Zeitschlitz (z. B. GSM mit 8 Zeitschlitzen).
  • Jedes Signal wird über eine eigene Lichtwellenlänge in der Faser übertragen.
  • Jedes Signal bekommt dauerhaft ein eigenes Frequenzband.
  • Alle senden gleichzeitig, getrennt durch einen Spreizcode.
Lösung

TDM teilt nach Zeit: gleiche Frequenz, jedes Signal bekommt einen festen, rotierenden Zeitschlitz. Medium: Kupfer/Funk. Beispiele: ISDN, GSM (8 Zeitschlitze pro Träger), DECT, Satellit.

12Multiple ChoiceAuto

Welches Medium nutzt WDM (Wellenlängenmultiplex)?

  • Glasfaser – mehrere Lichtwellenlängen in einer Faser.
  • Kupfer-Twisted-Pair mit mehreren Aderpaaren.
  • Koaxialkabel mit mehreren Zeitschlitzen.
  • Funk mit mehreren Frequenzbändern.
Lösung

WDM teilt nach Lichtwellenlänge (Farbe) — das optische Pendant zu FDM: mehrere Lichtwellenlängen in einer Glasfaser. Beispiele: DWDM-Backbone (>80 Kanäle pro Faser), Seekabel. Ideal in explosionsgefährdeten Umgebungen oder bei starker elektromagnetischer Beeinflussung.

13Multiple ChoiceAuto

Bei welchem Verfahren senden alle gleichzeitig auf derselben Frequenz, getrennt nur durch einen eindeutigen Code (z. B. UMTS, GPS)?

  • FDM (Frequenzmultiplex)
  • TDM (Zeitmultiplex)
  • CDM (Codemultiplex)
  • WDM (Wellenlängenmultiplex)
Lösung

CDM teilt nach Code: alle senden gleichzeitig auf derselben Frequenz, jedes Signal mit eigenem mathematischem Spreizcode; der Empfänger filtert per Code. Medium: Funk. Beispiele: UMTS (3G), GPS.

🔬 [Vertiefung] Bluetooth

🔬 [Vertiefung] – über den Kerninhalt hinaus: Piconet und Scatternet.

15Multiple ChoiceAuto

Wie viele aktive Slaves kann ein Bluetooth-Piconet neben dem einen Master maximal haben?

  • 1 aktiver Slave
  • 7 aktive Slaves
  • 15 aktive Slaves
  • 8 aktive Slaves
Lösung

Ein Piconet besteht aus 1 Master + bis zu 7 aktiven Slaves (FHSS-Frequenzsprungverfahren). Mehrere Piconets ergeben zusammen ein Scatternet.
